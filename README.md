# Antigravity Feishu Bot (飞书大模型私人助理)

这是一个基于飞书原生 WebSocket 接口和本地 `antigravity` AI 引擎深度结合的**全能型个人助理机器人**。

经过重构升级，本机器人已具备市面上顶级商业 AI 助手的全部核心体验，并独享基于本地服务器的“外挂”级系统权限。它可以作为你专属的生产力多面手，全天候 24 小时运行。

---

## 🌟 核心能力全景图与详细功能清单

本机器人的功能深度远超常规的 API 壳子，它深度接管了操作系统权限与多模态流媒体，具体支持以下核心维度：

### 一、 💬 极致交互体验 (UX & UI)
1. **真实打字机流式卡片 (Debounced Streaming UI)**
   摒弃传统的“转圈死等”，采用异步 Token 桶防抖机制（1.0s 强制锁频刷新）。大模型的每个字符都会在飞书互动卡片中如打字机般丝滑输出，彻底杜绝触发飞书限流阈值。
2. **精美状态预告卡片流转**
   处理超大体积的文件时，拒绝死寂。它会立刻下发蓝色的 `📥 资源加载中...` 卡片，精准汇报当前正在下载的文件名；当下载成功并进入分析阶段时，卡片会自动无缝变身为 `✨ AI 思考中...`，提供满满的安全感。
3. **沉浸式卡片视觉设计**
   摒弃简陋的纯文本，采用 `CardBuilder` 静态引擎渲染卡片。不同状态拥有专属页眉色彩（天蓝色提示、红色报错），并在页脚常驻显示：`当前使用的模型`、`当前佩戴的角色面具` 以及 `精准到秒的时间戳`。
4. **原生多选交互按钮**
   如果遇到需要用户进行“A/B/C”选择的分支问题，机器人会自动在卡片底部生成原生按钮（Button），点击按钮瞬间完成选择，无需手动打字回复。

### 二、 👁️ 全模态超感官引擎 (Multi-modal)
无论你向它扔什么文件，它都能“嚼碎了”吃进去：
1. **长文档深度阅读**：直接在聊天框发送 Word / PDF / TXT / Markdown 附件，它会在本地秒级下载，提取数万字内容后根据你的提问进行精准归纳与回答。
2. **“看图说话”**：发一张截图或照片，模型能瞬间理解画面内容，甚至可以直接读出图片上的代码并为你纠错。
3. **影音视听能力**：随手发一段几十兆的 `.mp4` / `.mov` 视频，或者长达几十分钟的 `.ogg` 录音文件，机器人能自动执行下载解析，从视音频源文件中提取深层信息为你生成纪要（依赖 Antigravity 底层多模态接口）。

### 三、 💻 赛博分身与系统级掌控 (System Agent)
借助底层 `antigravity` 引擎，你的飞书化身为了你私人 Mac/服务器 的高级可视化终端：
1. **远程文件操作**：你可以直接对它说“帮我看看桌面上 `project.py` 里的代码写了啥”，它会自动通过系统权限读取文件并发送给你。
2. **终端指令执行器**：你可以命令它“帮我运行一下 npm run build 看看报错原因”，它会实打实地去运行 Shell 指令。
3. **全自动实体物产出拦截**：当大模型通过代码为你画了一张趋势统计图（`.png`），或者生成了一个 Excel 表格时，底层的“雷达拦截器”会自动捕获这些系统新产生的文件，并将它们转化成飞书附件，直接反向发送到你的聊天框！

### 四、 🕹️ 全方位 Slash 指令台 (Commands)
完全协程化处理并发指令，目前内置了以下核心控制台操作：
* `/model`：弹出卡片式控制面板，提供按钮供你一键无缝切换后台大模型（支持如 Gemini, Claude, Llama 等）。
* `/role <身份>`：面具系统。让机器人死心塌地沉浸在你设定的身份里（如：`/role 资深前端架构师`），不再给出废话套话。
* `/remember <喜好>`：永久偏好刻录。比如输入 `/remember 以后所有代码都用 TypeScript 写`，该记忆会永久保存在 SQLite 数据库中，跨越无数次清空历史，始终生效。
* `/memory` 和 `/forget`：随时查看你当前刻录的所有长时偏好，或者一键全部清空。
* `/clear`：当上下文太长影响思考时，一键切断历史记忆，换用全新的 UUID 开始聊天。
* `/stop`：急停开关。当你发现大模型跑偏了，或者正在疯狂生成不需要的代码时，一键发送即可精准杀死后台对应的 LLM 僵尸子进程，省下大量算力。

## 🚀 安装部署指南

本机器人主要使用 Python 编写，并通过 PM2 守护进程保证 24 小时在线。

### 第一步：安装前置系统依赖
确保你的 Mac/Linux 系统中已安装以下工具：
1. **Python 3.10+** (推荐)
2. **Node.js 和 npm** (用于安装 PM2)
   ```bash
   npm install -g pm2
   ```
3. **安装 Lark CLI (飞书命令行工具)**
   `lark-cli` 用于处理多模态文件的下载和极个别系统交互。
   * 下载并安装: `npm install -g @larksuiteoapi/lark-cli`
   * （或者按飞书官方教程下载二进制版）
   * **重要**：安装后在终端运行 `lark-cli auth login` 完成飞书开发者授权登录。
4. **安装 Antigravity CLI**
   这是底层驱动大模型的引擎。你需要确保在终端中可以直接执行 `antigravity` 命令。

### 第二步：配置 Python 虚拟环境与依赖
为了不污染全局环境，建议在项目目录内使用虚拟环境 (venv)。

```bash
# 1. 克隆/进入项目目录
cd antigravity-feishu-bot

# 2. 创建 Python 虚拟环境
python3 -m venv venv

# 3. 激活虚拟环境
# Mac/Linux:
source venv/bin/activate

# 4. 安装核心依赖包
pip install -r requirements.txt
# (如果没有 requirements.txt，可以直接执行：pip install lark-oapi aiohttp)
```

### 第三步：配置飞书应用凭证
在项目根目录创建一个 `.env` 文件，填入你自己在“飞书开发者后台”创建的机器人的 `APP_ID` 和 `APP_SECRET`：

```env
APP_ID=cli_xxxxxxxxxxxx
APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxx
```
> **注意**：请确保你的飞书应用在后台开启了“WebSocket 建立长连接”的权限，并为机器人开通了收发消息的全部所需权限（包括 `im:message` 等）。代码底层已接入 `python-dotenv` 自动加载环境变量，避免了秘钥泄露至代码仓库的风险。

### 第四步：一键启动后台服务

完成配置后，使用 PM2 启动机器人即可让其在后台静默运行，无需保持终端开启。

```bash
# 启动机器人进程（请务必使用虚拟环境里的 python3 执行）
pm2 start venv/bin/python3 --name "feishu-bot" -- main.py

# 保存 PM2 进程列表，保证电脑重启后自动恢复
pm2 save
```

### 常用运维指令
* **查看实时日志**：`pm2 logs feishu-bot`
* **停止机器人**：`pm2 stop feishu-bot`
* **更新代码后重启**：`pm2 restart feishu-bot`

## 👨‍💻 架构原理解析

本代机器人的核心重构架构：
1. **网络层**：摒弃轮询，使用 `lark_oapi.ws.Client` 与飞书网关建立稳定的全双工长连接，事件响应延迟达到毫秒级。
2. **高度模块化解耦**：项目划分为 `main.py`（WebSocket 主轴）、`commands.py`（指令路由器）、`lark_client.py`（飞书 SDK 发送端）、`database.py`（存储层）以及 `multimodal.py`（多模态解析层），极致清爽。
3. **SQLite 持久化数据热迁移**：彻底告别脆弱的 JSON 文本文件读写，所有长时偏好记忆和对话上下文均无缝对接至原生的 SQLite 数据库，保证高并发数据安全。
4. **防僵尸与优雅退出机制**：引入 `signal` 信号流，接管中断（SIGINT/SIGTERM）。当使用 PM2 重启或者停止服务时，会自动执行 Graceful Shutdown，彻底清理后台任何遗留未处理的僵尸进程。

现在，尽情享受你的终极私人助理吧！遇到问题可以随时翻阅 `~/.pm2/logs/feishu-bot-error.log` 进行排查。
